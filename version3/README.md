## Security Model (v3.0)

| Attack Vector | Mitigated | How |
|---------------|-----------|-----|
| Forge TX as another user | ✅ | Ed25519 sig required; pubkey registered in ledger |
| Negative amount / mint coins | ✅ | Positive integer check + self-transfer block |
| Modify payload in transit | ✅ | Inner SHA-256 hash + Ed25519 sig both verified |
| Replay old TX | ✅ | Persistent monotonic seq; ledger rejects duplicate (sender, seq) |
| Malformed packet DoS | ✅ | All field access wrapped in try/except |
| Queue poison pill | ✅ | Broken files quarantined as .error |
| Race condition crash | ✅ | threading.Lock on reassembly dict |
| Float serialization mismatch | ✅ | Integer timestamps only |
| Fragment failure wastes bandwidth | ✅ | Abort remaining on permanent failure |

## Step-by-Step Execution

### STEP 0: Meshtastic Setup (One Time)

```bash
meshtastic --set lora.region US
meshtastic --set lora.modem_preset LONG_RANGE
meshtastic --info    # Record node IDs
```

Set Node B role:
```bash
meshtastic --set device.role ROUTER
```

Verify mesh connectivity: send test from A, confirm C receives via B.

---

### STEP 1: Generate Keys (One Time)

```bash
python mars_keygen.py
```

Distribute:
- `keys/user1.key` → Node A
- `pubkeys.json` → Node C

---

### STEP 2: Start Ledger (Node C, Terminal 1)

```bash
python ledger.py
```

Verify: `http://localhost:5000/ledger` → user1=1000, user2=0

---

### STEP 3: Start Gateway (Node C, Terminal 2)

```bash
python mars_recv.py
```

---

### STEP 4: Disconnect Node A

```bash
# macOS
networksetup -setairportpower en0 off
# Linux
nmcli radio wifi off
# Windows
netsh interface set interface "Wi-Fi" disable

# Verify
ping -c 1 8.8.8.8   # Should fail
```

---

### STEP 5: Send Transaction (Node A)

```bash
python mars_send.py --to user2 --amount 100 --dest '!nodeC_id'
```

Expected: TX signed, fragmented into ~15 packets at SF12, sent over LoRa.
On permanent fragment failure: remaining fragments aborted, user warned.

---

### STEP 6: Observe Node B

Watch OLED — fragments relay individually. Node B never holds full TX.

---

### STEP 7: Observe Settlement (Node C, Terminal 2)

Fragments arrive, reassemble, inner hash verified, signature verified,
ledger updated. Check: `http://localhost:5000/ledger`

---

### STEP 8: Reconnect and Reconcile (Node A)

```bash
networksetup -setairportpower en0 on   # macOS
python mars_verify.py --ledger http://<NodeC_IP>:5000
```
