## Changelog from v1.0

v1.0 was reviewed via adversarial simulation (Google AI Studio) and 7 critical
flaws were identified and fixed:

| # | Flaw | Severity | Fix in v2.0 |
|---|------|----------|-------------|
| 1 | Zero authentication — anyone can forge TXs | Critical | Ed25519 signatures on all TXs. Ledger verifies before settlement. |
| 2 | Orphaned queue — queue/ directory never flushed | Critical | Background thread retries queued TXs every 10 seconds. |
| 3 | Seq collision — 16.6 minute wraparound | High | Persistent monotonic counter file (seq_counter.json). Never wraps. |
| 4 | Reconciliation overwrite — dict key collision | Medium | Composite key (sender, seq) in mars_verify.py lookup. |
| 5 | Inner hash ignored — payload tamper undetected | High | Gateway verifies inner SHA-256 hash. Ledger verifies inner hash + signature. |
| 6 | No fragment retry — single failure kills TX | High | 3 retries per fragment with backoff. Failure tracking and user warning. |
| 7 | Protobuf overhead ignored — packets exceed PHY limit | Medium | 18-byte Meshtastic overhead subtracted from fragment budget. |

## Architecture

```
   Node A (OFFLINE)              Node B (RELAY)           Node C (ONLINE)
   ┌──────────────┐             ┌──────────┐            ┌──────────────┐
   │ Laptop       │             │ Heltec   │            │ Laptop       │
   │ WiFi OFF     │   915 MHz   │ V3 or    │  915 MHz   │ WiFi ON      │
   │              │ ──────────► │ Seeed    │ ────────►  │              │
   │ mars_send.py │             │ Solar    │            │ mars_recv.py │
   │ Heltec V3    │             │ Repeater │            │ Heltec V3    │
   │ USB serial   │             │          │            │ USB serial   │
   │              │             │ Standard │            │ ledger.py    │
   │ USER 1       │             │Meshtastic│            │ (Flask)      │
   │ Ed25519 key  │             │ firmware │            │ pubkeys.json │
   └──────────────┘             │ NO CODE  │            └──────────────┘
                                └──────────┘
```

**Security model:** Node B relays encrypted fragments it cannot read, forge,
or tamper with. Even if Node B is compromised, it cannot:
- Forge transactions (no signing key)
- Modify payloads (inner hash + signature breaks)
- Replay transactions (monotonic seq rejected as duplicate)

## Step-by-Step Execution

### STEP 0: Meshtastic Setup (One Time)

Flash all 3 devices with Meshtastic. Set same channel:

```bash
meshtastic --set lora.region US
meshtastic --set lora.modem_preset LONG_RANGE   # Forces SF12
meshtastic --info   # Record node IDs
```

Set Node B as a router:
```bash
meshtastic --set device.role ROUTER
```

Record node IDs:
```
Node A: !__________
Node B: !__________
Node C: !__________
```

Verify mesh: send test message from A, confirm C receives it via B.

---

### STEP 1: Generate Keys (One Time, Either Laptop)

```bash
cd mars-poc
python mars_keygen.py
```

Output:
```
  user1: keys/user1.key
    pub: 3a7f2b9c1d4e8f0a...
  user2: keys/user2.key
    pub: 9b2c4d6e8f1a3b5c...

  Public keys written to pubkeys.json
```

**Distribution:**
- Copy `keys/user1.key` → Node A laptop (mars-poc/keys/)
- Copy `pubkeys.json` → Node C laptop (mars-poc/)
- `ledger.py` on Node C reads `pubkeys.json` to verify signatures

more fragments = more visible relay activity on Node B.

## Security Model

| Attack | v1.0 | v2.0 |
|--------|------|------|
| Forge transaction | ❌ Trivial — no auth | ✅ Blocked — Ed25519 sig required |
| Modify amount in transit | ❌ Inner hash ignored | ✅ Blocked — inner hash + sig verified |
| Replay old TX | ❌ Seq wraps in 16 min | ✅ Blocked — monotonic counter, never wraps |
| Drain via Node B | ❌ No auth needed | ✅ Blocked — Node B has no signing key |
| Queue data loss | ❌ Queue never flushed | ✅ Fixed — background flush every 10s |
| Exceed PHY MTU | ❌ Protobuf not accounted | ✅ Fixed — 18B overhead subtracted |
